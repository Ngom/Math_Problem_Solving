21 Question 21

Language: Intermediate Student with Lambda

Preparation: Module 8

In this question, you will complete the implementation of Braun trees in Racket that was started in lecture.

Write brt-extend (consuming a Braun tree and a new element, and producing a Braun tree) and brt-tail (consuming a Braun tree, and producing a Braun tree). brt-tail should raise an error containing "empty" if it is applied to an empty Braun tree.

Then write brt-update, which consumes a Braun tree t, an index i, and an element elt, and produces the Braun tree representing the sequence that t represents, except that the element of index i is elt. The efficiency of brt-update should be the same as brt-index, and brt-update should produce an error containing the word "range" if the index is out of range. All your code should use structural recursion on Braun trees.

To submit: Q21.rkt.

Notes:

If you have programmed in other languages and have used arrays, your correct solutions to this question demonstrate that a purely functional data structure can simulate an array with a multiplicative overhead that is logarithmic in the size of the array. Arrays are present in full Racket, where they are called vectors.
