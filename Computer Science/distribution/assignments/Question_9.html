9 Question 9

Language: Beginning Student.

Preparation: Module 5, through slide 23.

Write the Racket function unique-left, which consumes a list of elements and produces a list which has all but the leftmost occurrence of each value removed. For example, if unique-left is applied to (list 1 4 2 1 5 4), the result would be (list 1 4 2 5). The list may contain different types of comparable values, not just numbers.

Then write the Racket function unique-right, which consumes a list of elements and produces a list which has all but the rightmost occurrence of each value removed. That is, the result of applying it to (list 1 4 2 1 5 4) would be (list 2 1 5 4). Your code for unique-right should not use unique-left; treat these as independent problems (though they may share helper functions).

Use structural recursion on lists for both functions and for all recursive helper functions. Use only function definition, function application, cond, empty?, cons, cons?, first, rest, and equal? to answer this question. In fact, if you use the template for structural recursion on lists, which includes one cond, one empty?, and one cons?, you do not need additional uses of these.

To submit: Q9.rkt.

Notes:

Your functions should have the property that on any given application, there is at most one recursive application. If two or more recursive applications are possible in some cases, then your code may run too slowly for some arguments. You may need to write extra helper functions to ensure this, by making the single recursive application an argument to a helper function (so that it is evaluated once as the argument, but may be used several times in the helper function). Later, we will see how to use local definitions to avoid this awkwardness. Subsequent questions will not repeat this instruction, but you should remember and apply it.

Do not use the built-in reverse or remove functions, and do not write your own identical versions of these. If a function you write for this question seems to produce its result in reverse order, you can usually rewrite its code to produce the correct result. The built-in remove function only removes one occurrence of a value from a list, and using it as a helper function leads to non-structural recursion. It is okay to write something similar whose use preserves structural recursion.
