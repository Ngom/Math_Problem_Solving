14 Question 14

Bonus challenge: This question is somewhat more difficult and/or time-consuming, so it is not part of the regular course requirements. You may work on it with other students, and you do not need to submit it for credit.

There is one more similar bonus challenges among later questions. Completing this question will help you with the later bonus question. Completing Question 13 will help you with this question. Completing all three of these bonus questions will give you excellent practice in structural recursion on lists, and I highly recommend it, if you have the time.

Language: Beginning Student with List Abbreviations

Preparation: Module 5

Write the Racket function comb (for combinations) that consumes a list lst representing a set using the ordered list representation, and a natural number n, and produces a list of ordered lists representing the size-\(n\) subsets of lst. For example, when comb is applied to '(1 2 3) and 2, it might produce '((1 2) (1 3) (2 3)). As in Question 13, the order of the sub-lists in the result list is not important.

Use structural recursion on one list parameter and one natural number parameter; think carefully about the base cases, including their ordering. Do not use your solution to Question 13; you should not generate all subsets. Further, you should not generate any subsets of size greater than \(n\), you should not use the built-in length function or write your own, and you should not need any arithmetic except for what is used in the template for structural recursion on natural numbers.

To submit: Q14.rkt.
