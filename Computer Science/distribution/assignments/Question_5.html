5 Question 5

Bonus challenge: This question is somewhat more difficult and/or time-consuming, so it is not part of the regular course requirements. You may work on it with other students, and you do not need to submit it for credit.

Language: Beginning Student.

Preparation: Module 4, through slide 11.

Write the function in-triangle that consumes four point structs p1, p2, p3, and p, representing points in the Euclidean plane with rational coordinates, and produces a Boolean value which is true if and only if the point p is on or within the (possibly degenerate) triangle defined by p1, p2, and p3.

The point structure is defined early in Module 3 using the following definition, which you should include in your code:

(define-struct point (x y))

There are many possible ways to answer this question mathematically, and you are permitted to use the Internet to research them (though you may enjoy working them out yourself). Keep in mind that Racket can represent rational numbers with arbitrarily large numerators and denominators, and your function must be correct for all arguments. Consequently, you should avoid computations that produce inexact values (e.g. square roots or trigonometric functions) as they may result in errors.

You should make effective use of helper functions. Include in comments a high-level description of the computation, in addition to the design recipe elements.

To submit: Q5.rkt.

Notes:

This question is harder than previous ones because there are two levels of complexity. The first is in working out or understanding a mathematical solution and translating it into Racket code. But the second arises from computational difficulties that tend to be ignored in mathematical treatments. A mathematical solution is often described using a "nicely drawn" triangle, and this is not guaranteed here.

We already saw above that using inexact numbers may lead to errors. Some of the tests in the testing program are designed to cause code relying on inexact numbers to fail. Some solutions compute slopes of lines. Will they still work when two of the points defining the lines have the same x-coordinate?

Finally, the words "possibly degenerate" in the question indicate that two or more of the points may be the same. This definitely does not result in a "nicely drawn" triangle, but the triangle is still defined, and your code must still produce an answer.
